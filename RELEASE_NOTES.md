- **Phase 1 status**: all six tickets (`rao-001`–`rao-006`) are now closed in `backlog/done/`; `backlog/specification/` is empty. Two requirements carried forward to future tasks: a build-time test asserting the Mock AI Provider makes zero outbound network calls (Phase 12), and a v1→v2 vendor/DPA review gate before a real LLM provider ships (`docs/PRODUCT-ROADMAP.md`).
- **rao-007**: Phase 2 (Core Technical Architecture) completed and closed — new `docs/PHASE2-CORE-TECHNICAL-ARCHITECTURE.md` adds Service-Oriented Architecture conventions, a SOLID-principles mapping, Agent Engine internals (Registry/Lifecycle/Runner + concurrency model), a Workflow Engine design, a concrete Event Bus (built on `ActiveSupport::Notifications`), Conversation/Memory/Prompt architecture, and all ten cross-cutting engineering strategies. Three requirements carried forward to future implementation tasks: Event Bus subscribers must be non-blocking, the Concurrency Guard must be atomic (not check-then-act), and the dependency-graph cache needs insert+delete invalidation test coverage.
- **rao-008**: Phase 3 (Mock AI Provider Foundation) completed and closed — new `docs/PHASE3-MOCK-AI-PROVIDER-FOUNDATION.md` defines the Provider Interface (request/response/error/capability/configuration models), the Mock AI Provider's architecture and lifecycle, Provider-specific Conversation/Agent Execution flow detail, Prompt Management and the 11-category Prompt Template Library, the Mock Response Strategy (12 deterministic scenarios), Token/Cost Simulation, and the Future Migration Plan's mechanics. A pre-close-out revision pass caught and fixed a genuine correctness bug (multiple `tool_calls` from one turn would have collided on a shared idempotency key, silently dropping records) plus three completeness gaps (missing `memory_updates` field, non-round-aware fixture selection, under-specified `latency_ms`). One requirement carried forward: real providers must require validated, non-nil credentials before activation. All eight tickets across Phase 1, 2, and 3 are now closed in `backlog/done/`.
